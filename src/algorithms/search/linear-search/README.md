# Линейный поиск

Алгоритм поиска элемента в массиве или списке. Он последовательно проверяет каждый элемент, начиная с первого, пока не найдет искомый элемент или не достигнет конца массива.

## Принцип работы

1. Начинается с первого элемента массива;
2. Каждый элемент сравнивается с искомым значением;
3. Если элемент найден, возвращается его индекс;
4. Если массив пройден полностью, а элемент не найден, возвращается специальное значение (например, -1).

## Временная сложность

- В худшем случае: O(n) (когда искомый элемент находится в конце массива или отсутствует);
- В лучшем случае: O(1) (когда искомый элемент находится в начале массива);
- Средняя сложность: O(n).

## Пространственная сложность

- O(1), так как дополнительная память не требуется.

## Преимущества

- Простота реализации;
- Работает с любыми типами данных (числа, строки, объекты и т.д.);
- Не требует предварительной сортировки массива.

## Недостатки

- Низкая производительность на больших массивах;
- Неэффективен для повторяющихся поисков в одном и том же массиве.
